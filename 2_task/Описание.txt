Описание к Postman-коллекции

На уровне коллекции есть переменные:
BaseURL - чать URL, которая присуствует в каждом запросе
id - id видео на YouTube из задания
key - ключ, который необходим в 1м и 3м методе для авторизации. Чтобы его получить, нужно привязать профиль разработчика к своему профилю google. Далее в консоли разработчика https://console.cloud.google.com создать проект и учетные данные клиента. Сервис выдаст ключ.
AccessToken - токен, который нужен во 2м методе для авторизации. Его можно получить тут https://developers.google.com/oauthplayground/?code=4/0AfJohXkZhS8oTJHRdFlIHsQ2HwsWIsz-zpgkusdAegFKpZWoFMBkbdeqonEvbL0iYLEilg&scope=https://www.googleapis.com/auth/youtube%20https://www.googleapis.com/auth/youtube.readonly
Нужно обменять код авторизации на токен, разрешив доступ к своему аккаунту. Система выдаст токен доступа. Он будет годен 1 час, но его можно обновить.

Позитивные кейсы:
1. Получение списка видео.
В параметрах метода обязательно должны указываться ключ для авторизации и id видео.
Есть проверка на статус-код 200
2. Запрос, с помощью которого можно поставить лайк к видео.
В параметрах обязательно указывается id видео, к которому нужно поставить лайк, рейтинг, в котором указывается, что мы хотим поставить именно лайк, а не дизлайк. В авторизации указываем способ OAuth 2.0 и указываем токен доступа. В случае успеха в ответе нет тела. Присутствует проверка на статус-код 204.
3. Получение списка популярных видео.
В параметрах метода обязательно должны указываться ключ для авторизации и параметр chart, указывающий, фильтр (популярные видео). Есть проверка на код 200.

Негативные кейсы:
1 кейс проверяет 1-й метод без авторизации
2 кейс проверяет 1-й метод без обязательного параметра id
3 кейс проверяет 2-й метод без авторизации
4 кейс проверяет 2-й метод без обязательного параметра rating
5 кейс проверяет 3-й метод без обязательного параметра chart
6 кейс проверяет 3-й метод без без авторизации
